Doing research in the 21st century requires good IT infrastructure. The purpose of this article to show you how I build my home lab environment.

Function List

  • File Station
    • Important personal files, like photos. Requires backup solution
    • Backup server for my computers
    • Multiple user support
    • Media center
  • Programing tools
    • Rstudio (through docker)
    • IPython Notebook (through docker)
    • Data front-end
  • Downloading Station
    • Low energy consumption
    • Stable
  • VPN Server
  • Remote manipulation capacity

Hardware

As a budget-efficient solution, my current home lab consists:

  • Linksys EA4300 Router
  • WD MyCloud 4T Edition
  • Pogoplug
  • Terminals:
    • Dell Optiplex 9020 SFF
    • Thinkpad X250
    • Apple Macbook Air (2014)
    • Gigabyte Brix
    • Sony Playstation 4
    • Dell Chromebook 11

Intranet

The Linksys Router is center of my home lab. It is wire connected to Dell Optiplex, which is my main workstation, Brix, which is my wifi’s main workstation, and WD MyCloud, for faster file transfer. I also wire connect the PogoPlug, whose role I will explain later. All wire connected devices are assigned fixed IP.

Other devices use wireless network.

Now every terminal can access the files on WD MyCloud. Further, I enabled VNC and wake on lan for both workstations so that I can remote on any of them when needed.

Accessing from outside

So I use the Linksys Router as firewall, managing which port to expose. The key player linking intranet and internet is PogoPlug. More specifically:

  1. I use ddclient to update my IP address to the binded url address
  2. PogoPlug runs a VPN server. After which I am able to use services which are only exposed to local network.
  3. PogoPlug is always on, and can be used to wake on lan other workstations remotely. Since PogoPlug uses ARM chipset, the energy consumption is greatly reduced.
  4. I sometimes SSH into the network to use VNC.
