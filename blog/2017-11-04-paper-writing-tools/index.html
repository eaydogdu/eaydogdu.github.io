"Swiss Knife" for Paper Writing

Publish Date: Nov 4, 2017    Last Update: Jun 28, 2018

Table of Contents

  • First best
  • The Swiss Knife
  • Conclusion

While handy tools could boost paper-writing efficiencies, searching for such tools is usually a time-consuming process. This post documents my workflow for academic writing. It is based on years of trials and errors, and hopefully can provide readers with a good starting point to develop their own workflows.

First best

The first best setup should satisfy my following needs:

  1. Project management
    • to-do list
    • track and manage issues/comments/ideas
    • version control
  2. Text editing environment that features
    • beautiful interface for editing
    • powerful grammar check
    • easy collaboration
    • flexible format conversion (pdf, html)
  3. Bibliography management
    • powerful text search
    • tagging system
    • notes taking on pdf files

The Swiss Knife

I use a variety of software and services during the process. The raw ingredients include

  1. Paperpile - Bibliography management tool
  2. Github - Project management
  3. Vim + Markdown - Writing environment
  4. Pandoc - Conversion tool

So this is how it works. For each project, I will create a Github repository for code and manuscript. Github provides three great features that are especially suitable for academic writing: project, issue, and version control. I use the project feature to manage ToDos and track progress. For comments, problems, and improvement ideas, I use the issue feature to jot down these points. Such a system is also suitable for collaboration. Git is particularly suitable to track all changes and manager versions. Fixing possible conflicts in codes or manuscripts is also simple with Git. With Github, we can further assign each issue or Todo to different collaborators. Overall, a Github repository serves as a solid foundation for a project.

I edit texts in Vim. While the learning curve for Vim is quite steep at the beginning. However, once you are used to the workflow and accumulate a set of extremely handy plugins, you will get addicted to the idea that your hands never need to leave the keyboard. Needs for grammar check, beautiful interface, power editing functions can all be achieved through vim plugins.

I write the manuscripts mostly in Markdown, which is a markup language which features high readability and syntax simplicity. Using pandoc, a format conversion tool, you can easily turn markdown files in to .docx, .html, or .pdf files. In this way, we can focus on the contents first and later convert the source file to different formats as needed.

Finally, I use Paperpile to manage bibliography. This is the only service that requires a subscription, but the quality of the service is totally worth the money. The coolest feature is that Paperpile saves all the PDF files in Google Drive, and you can annotate the files as well. Everything is saved and synced on the cloud, and almost all texts, including our annotations, are searchable. For each project, I would use a specific tag on all the references. Then I can easily export a .bibtex file containing all the papers with the tag.

Conclusion

That’s it guys! Of course, everyone has his/her own opinions about an efficient workflow. Hopefully this post contains some useful information for anyone who is in the process of workflow development. But remember, in the end it is contents of a paper that count, not the tools for writing it!

Happy writing great paper!
