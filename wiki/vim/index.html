Basic

Workflow

I sync the whole .vim folder in github. I use pathogen to manage all the plugins.

Most Basic

For basic usually, we have

# Navigation
(h, j, k, l) character navigation
(w, e, b) word navigation
(%) go to matching parentheses
(0, $) go to the beginning/end of a line
(*, #) find the next/prev word under cursor
(g, 3G) gg to the beginning, G to the end. 3G to the 3rd line
(/) to search, n for next result, N for prev result
(x, r) x to del a charater, and r to replace a character
(.) to repeat last action

# Quick Formatting
ggVG=

Plugins

I use Pathogen and Git to manage all my plugins. Such can be done as

cd ~/.vim/bundle
git clone git://github.com/tpope/vim-xx.git
cd ~/.vim
git submodule init
git submodule add git://github.com/tpope/vim-xx.git bundle/vim-xx
git commit -m "Added vim-xx"
git push

In other computer, to download all configurations and plugins, just use

git clone git@github.com:lipop/vim.git ~/.vim
cd ~/.vim
git submodule init
git submodule update

To update the submodules, use

git submodule foreach git pull origin master
  1. Pathogen: 管理所有的插件，配合Github，所有的插件也都备份到Github中
  2. Vimwiki: 用来管理个人的wiki。利用pandoc重新写了转换的文件
  3. NERDTree：用来配合文件查看路径
  4. Syntastic：用来查找语法错误
  5. SimpyFold: To fold the python codes
  6. Vim-markdown: To highlight the markdown files

Tricks

General

  1. 分屏技巧
:new \\ 建立一个新的空白栏
:split \\ 将现有文件分两栏
:vnew :vsplit \\ 将窗口左右排布
ctrl+w \\ 选择窗口
:qall \\如果是运用到所有窗口的命令要在后面加all

:3split \\ 控制新的分屏高度为三行
ctrl+w + \\ 增加当前窗口的高度
:only \\只保留一个窗口
  1. 拼写检查
:set spell \ nospell  \\ 打开或关闭拼写检查      
]s \\ 下一个单词拼写错误    
[s \\ 上一个单词拼写错误    
zg \\ 将单词加入词典    
zug \\ 撤销将单词加入词典   
z= \\ 拼写建议

Special Cases

  1. gVim的一些设置 在安装了gVim之后，使用上出现了两个问题：一是在启动的时候总是出现recovery无法 找到的错误，二是无法使用退格键（设置了indent方式的前提下）。所以我的解决办法是：
set directory=.,$TEMP
set backspace=indent,eol,start
  1. 输入^M
    有时候从Windows下面建立的文件在Linux里面打开时，会在每行的末尾出现^M，处女座 座的同学表示看着蛋疼。Vim下解决办法很简单，替换就好。难点是^M的输入有点技巧。 按住Ctrl+V, 松开V，再按M，即可。
