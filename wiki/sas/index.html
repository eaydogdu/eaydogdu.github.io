General principals

  1. Each line ends with ;
  2. It does not distinguish from cap letters from lower letters
  3. Only two data types: numeric and character

Data

Data Import

Requirements for list input are:

  • Missing values are replaced with .
  • Characters can not contain space, less than 8
  • Observations are separated by space
* Create a sample dataset ;
DATA sample ;
  INFILE '/home/sample.dat' ;
  INPUT Var1 $ Var2 ;
  * Character variable with $ after ;
* Print the dataset with PROC Step ;
PROC PRINT DATA = sample ;
  TITLE 'Sample dataset' ;
RUN ;

Overcoming these requirements are column input. Sample codes are

INPUT var1 $ 1-10 var2  11-13 var3 14-17 ;
  * The first variable takes 1-10th column ;

For non-standard data, like date or strings with $ sign we can use the informats way:

INPUT Name $16. Age 3. Date MMDDYY10. ;

More advanced data input methods include

  • Using mixed method (with @)
  • Reading messy original data
  • Reading through multiple lines
  • Reading multiple observations in one line

Data Summary

  • Show the description of the data sas proc contents data = DATA ; run ;
  • Show the summary statistics of the data sas proc means data = DATA ; var VAR1 VAR2 ; run ;
  • Print the data sas proc print data = DATA (firstobs = 1 obs = n) ; run ;
