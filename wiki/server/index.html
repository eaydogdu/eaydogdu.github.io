I personally host a number of servers. I mainly use these servers as file station, downloading station, or as production environment. The benefits of using servers include running programs for days, or constantly hosting services. This entry documents server setup, maintenance, and service setup.

Server Setup

Choosing a stable Linux distribution is important. I personally recommend Ubuntu server or Arch Linux.

The installation follows usual practice. After installation, important things include:

  1. Add user and disable root from SSH
  2. Set up non-password SSH keys
  3. Use tools like ufw to manage port expose

Personally, I only keep the SSH port open. To use other service, I would use tunnels through SSH. For example, if you want to use VNC, so you map the server port to your local host port through SSH

ssh -L rPort:localhost:lPort -N -f -l user server.ip
  1. -N stands for just port forwarding
  2. -f stands for background
  3. -l states the user

Server Maintenance

Status Monitoring

One can check the status with following shell commands

  uptime    # show the running time
  df        # check disk usage
  free      # check memory usage
  cat /proc/cpuinfo   # check cpu info

To check the current load

  top       # show current loading
  ps aux    # show all processes
  uname -a  # see distribution info
  who       # see logged on users

One can also consider graphical monitoring tools. For small-load or personal-use servers, these tools might be too much.

Security

Check currently loaded user

  w user    # check user
  who       # check login users
  last      # check login history

Services

Thanks to docker, setting up a production environment has not been easier. Detailed usage can be referred to the docker wiki. Of course you can use services directly on the server machine. A list of services we can run includes:

  1. Rstudio-server
  2. Website hosting
  3. IPython-notebook
  4. Transmission
